# MoldForge landing page

Static, self-contained landing page for MoldForge (intended for
**https://moldforge.plesuro.eu**).

```
site/
  index.html              # the whole page (inline CSS + minimal JS)
  assets/                 # hero + gallery renders (PNG)
  downloads/              # the add-on .zip + index.json (Blender extension repo)
```

## Deploy

It's a plain static site - no build step. Upload the contents of `site/` to
your web root so that:

- `https://moldforge.plesuro.eu/` serves `index.html`
- `https://moldforge.plesuro.eu/assets/…` and `/downloads/…` resolve

Any static host works (nginx/Apache document root, Netlify, Cloudflare Pages,
GitHub Pages, an S3 bucket, etc.). Just point it at this folder.

## Updating the download

When you ship a new add-on version:

1. Build the new zip straight into `downloads/` and remove the old one:
   `blender --command extension build --source-dir moldforge --output-filepath moldforge/site/downloads/moldforge-X.Y.Z.zip`
2. Regenerate the extension-repo index so the in-Blender updater sees it:
   `blender --command extension server-generate --repo-dir moldforge/site/downloads`
3. In `index.html`, update the two references in the Download section
   (the `href` and the version label `moldforge-X.Y.Z.zip · ~NN KB`).

## Auto-update (Blender extension repository)

`downloads/` is also a self-hosted Blender **extension repository** (`index.json`
generated by step 2 above), so users get the built-in **Check for Updates / Update**
button instead of re-installing by hand. Once deployed, a user adds it once:

- **Edit ▸ Preferences ▸ Get Extensions ▸ Repositories ▸ + ▸ Add Remote Repository**
- URL: `https://moldforge.plesuro.eu/downloads/index.json`
- Enable it, then **Check for Updates** — MoldForge installs/updates from there and
  flags new releases automatically.

The one-click Download button still serves the raw `.zip` for a manual install; the
repo is only for those who want automatic updates.

## Notes

- Reveal-on-scroll is progressive enhancement: with JavaScript off, everything
  is shown immediately (`.no-js` fallback), so it stays SEO/accessibility-safe.
- Colours/copy live at the top of `index.html` (`:root` CSS variables and the
  section markup) - easy to tweak.
- Replace the renders in `assets/` with your own product shots any time; keep
  the same filenames or update the `<img src>` paths.
